多线程在OC中的应用（GCD）

发表于 2016-11-26   |   分类于 objective-c   |     |   阅读次数

GCD 全称Grand Center Dispatch

是 Apple 开发的一个多核编程的解决方法，简单易用，效率高，速度快，基于C语言，更底层更高效，并且不是Cocoa框架的一部分，自动管理线程生命周期（创建线程、调度任务、销毁线程）。

GCD中的三种队列类型

GCD编程的核心是dispatch队列，dispatch block的执行最终都会放在某个队列去执行。

  • The main queue(主线程串行队列)：与主线程功能相同，提交至Main queue的任务会在主线程中执行。

  • Global queue(全局并发队列)：全局并发队列有整个进程共享，有高、中（默认）、低、后台四个优先级别。

  • Custom queue(自定义队列)：可分为串行和并行。

  • Group queue(队列组)：将多线程进行分组，最大的好处是可获得所有线程的完成情况。

The main queue(主线程队列)

dispatch_sync同步执行任务函数，不会开启新的线程，dispatch_async异步执行任务函数，会开启新的线程。

1、获取主线程串行队列

dispatch_queue_t mainQueue = dispatch_get_main_queue();

2、主线程串行队列执行同步任务，会产生死锁。程序一直处于等待状态，block中的代码将执行不到

dispatch_queue_t mainQueue = dispathc_get_main_queue()
dispatch_async(mainQueue,^{
    // do something
    NSLog(@"main queue");
});

3、主线程串行队列异步执行任务，不会产生死锁。程序正常运行，block中的代码正常执行。

dispatch_queue_t mainQueue = dispathc_get_main_queue()
dispatch_sync(mainQueue,^{
    // do something
    NSLog(@"main queue");
});

4、从子线程，异步返回主线程更新UI

dispatch_queue_t globalQueue = dispatch_get_global_queue(DISPATHC_QUEUE_PRIORITY_DEFAULT,0);
dispatch_async(globalQueue,^{
    // 子线程执行下载任务，防止主线程卡顿
    NSURl *url = [NSURL URLWithString:@"http;//www.xxxx.xxx.com"];
    NSError *error;
    NSString *data = [NSString stringWithContentsOfURL:url encoding:NSUTF8StringEncoding error:&error];
    if (data != nil) {
        dispatch_queue_t mainQueue = dispatch_get_main_queue();
         //异步返回主线程，更新UI
        dispatch_async(mainQueue, ^{
        NSLog(@"根据更新UI界面");
        });
    } else {
        NSLog(@"error when download:%@",error);
    }

});

主线程串行队列有系统默认生成，无法调用dispatch_resume()和dispatch_suspend()来控制执行继续和中断。

Global queue（全局并发队列）

1、获取全局并发队列

//DISPATCH_QUEUE_PRIORITY_DEFAULT,DISPATCH_QUEUE_PRIORITY_HIGH,DISPATCH_QUEUE_PRIORITY_LOW,TDISPATCH_QUEUE_PRIORITY_BACKGROUND
// 一般选择默认的DEFAULT
dispatch_queue_t globalQueue1 = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0);

2、全局并发队列同步执行任务，在主线程执行会导致页面卡顿。

dispatch_queue_t globalQueue = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0);
NSLog(@"current task");
dispatch_sync(globalQueue, ^{
    sleep(2.0);
    NSLog(@"sleep 2.0s");
});
NSLog(@"next task");

控制台输出如下：

 2016-11-26 11:07:23.350 gcd[2023:906461] current task
2016-11-26 11:07:25.352 gcd[2023:906461] current thread = <NSThread:         0x7fae31602f50>{number = 1, name = main}
2016-11-26 11:07:25.353 gcd[2023:906461] sleep 2.0s
2016-11-26 11:07:25.353 gcd[2023:906461] next task

可以看到，block中的当前线程是主线程，会先执行block中的代码，才会往后执行。

3、全局并发队列异步执行任务，在主线程运行，会开启新的线程去执行任务，页面不会卡顿。

dispatch_queue_t globalQueue = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0);
NSLog(@"current task");
dispatch_async(globalQueue, ^{
    sleep(2.0);
    NSLog(@"current thread = %@",[NSThread currentThread]);
    NSLog(@"sleep 2.0s");
});
NSLog(@"next task");

控制台输出如下：

2016-11-26 11:08:51.879 gcd[2048:908843] current task
2016-11-26 11:08:51.880 gcd[2048:908843] next task
2016-11-26 11:08:53.885 gcd[2048:908899] current thread = <NSThread: 0x7f9610e04170>{number = 2, name = (null)}
2016-11-26 11:08:53.885 gcd[2048:908899] sleep 2.0s

block中的线程是新开启的子线。

4、多个全局并发队列，异步执行任务。

dispatch_queue_t globalQueue = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0);
NSLog(@"current task");
dispatch_async(globalQueue, ^{

    NSLog(@"current thread = %@",[NSThread currentThread]);
    NSLog(@"最先加入队列");
});
dispatch_async(globalQueue, ^{

    NSLog(@"current thread = %@",[NSThread currentThread]);
    NSLog(@"其次加入队列");
});
NSLog(@"next task");

控制台输出：

2016-11-26 14:00:18.430 gcd[2213:1013033] current task
2016-11-26 14:00:18.431 gcd[2213:1013033] next task
2016-11-26 14:00:18.431 gcd[2213:1013245] current thread = <NSThread: 0x7fc531d2ecc0>{number = 2, name = (null)}
2016-11-26 14:00:18.432 gcd[2213:1013245] 其次加入队列
2016-11-26 14:00:18.431 gcd[2213:1013246] current thread = <NSThread: 0x7fc531e0b120>{number = 3, name = (null)}
2016-11-26 14:00:18.432 gcd[2213:1013246] 最先加入队列

异步线程的执行顺序是不确定的。

全局并发队列由系统默认生成，无法调用dispatch_resume()和dispatch_suspend()来控制执行继续和中断。

Custom queue（自定义队列）

1、自定义串行队列

  • 获取自定义串行队列

dispatch_queue_create(const char *label, dispatch_queue_attr_t attr)函数中第一个参数是给这个queue起的标识，这个在调试的可以看到是哪个队列在执行，或者在crash日志中，也能做为提示。第二个是需要创建的队列类型，是串行的还是并发的

dispatch_queue_t serialQueue = dispatch_queue_create("com.wintrerain5.serialQueue", DISPATCH_QUEUE_SERIAL);
NSLog(@"%@",serialQueue);

2016-11-26 14:19:07.134 gcd[2266:1032314] <OS_dispatch_queue: com.wintrerain5.serialQueue[0x7ff701c09550]>

  • 自定义串行队列同步执行任务

    dispatch_queue_t serialQueue = dispatch_queue_create("com.wintrerain5.serialQueue", DISPATCH_QUEUE_SERIAL);
NSLog(@"current task");
dispatch_sync(serialQueue, ^{
    NSLog(@"最先加入串行队列");
});
dispatch_sync(serialQueue, ^{
    NSLog(@"其次加入串行队列");
});
NSLog(@"next task");

控制台输出：

2016-11-26 14:26:45.013 gcd[2297:1040312] current task
2016-11-26 14:26:45.015 gcd[2297:1040312] current thread = <NSThread: 0x7fbe21401f20>{number = 1, name = main}
2016-11-26 14:26:45.015 gcd[2297:1040312] 最先加入串行队列
2016-11-26 14:26:45.016 gcd[2297:1040312] current thread = <NSThread: 0x7fbe21401f20>{number = 1, name = main}
2016-11-26 14:26:45.016 gcd[2297:1040312] 其次加入串行队列
2016-11-26 14:26:45.017 gcd[2297:1040312] next task

串行队列中先进来的线程先执行。

  • 自定义串行队列嵌套执行同步任务，产生死锁。

    dispatch_queue_t serialQueue = dispatch_queue_create("com.wintrerain5.serialQueue", DISPATCH_QUEUE_SERIAL);
dispatch_sync(serialQueue, ^{
    NSLog(@"会执行的代码");
    // 后面的代码都不会执行，程序被死锁在这里
    dispatch_sync(serialQueue, ^{
        NSLog(@"不会执行的代码");
    });
});

  • 异步执行串行队列，嵌套同步执行串行队列，同步执行的串行队列中的任务将不会执行。

2、自定义并发队列

  • 获取自定义并发队列

    dispatch_queue_t conCurrentQueue = dispatch_queue_create("com.wintrerain5.conCurrentQueue", DISPATCH_QUEUE_CONCURRENT);

  • 自定义并发队列执行同步任务

        dispatch_queue_t conCurrentQueue = dispatch_queue_create("com.wintrerain5.serialQueue", DISPATCH_QUEUE_CONCURRENT);
NSLog(@"current task");
dispatch_sync(conCurrentQueue, ^{
     NSLog(@"先加入队列 currentThread=%@",[NSThread currentThread]);
});
dispatch_sync(conCurrentQueue, ^{
    NSLog(@"次加入队列 currentThread=%@",[NSThread currentThread]);
});
NSLog(@"next task");

控制台输出：

2016-11-26 14:47:06.142 gcd[2360:1059122] current task
2016-11-26 14:47:06.143 gcd[2360:1059122] 先加入队列 currentThread=<NSThread: 0x7f8969c05770>{number = 1, name = main}
2016-11-26 14:47:06.143 gcd[2360:1059122] 次加入队列 currentThread=<NSThread: 0x7f8969c05770>{number = 1, name = main}
2016-11-26 14:47:06.144 gcd[2360:1059122] next task

任务从上至下依次执行。

  • 自定义并发队列嵌套执行同步任务。

     dispatch_queue_t conCurrentQueue = dispatch_queue_create("com.wintrerain5.serialQueue", DISPATCH_QUEUE_CONCURRENT);
NSLog(@"current task");
dispatch_sync(conCurrentQueue, ^{

不会产生死锁。

NSLog(@"current task");
dispatch_async(conCurrentQueue, ^{
    NSLog(@"先加入队列 currentThread=%@",[NSThread currentThread]);

    NSLog(@"次加入队列 currentThread=%@",[NSThread currentThread]);

异步执行任务，开启新的子线程，不影响当前线程任务的执行，并发队列中的任务，几乎是同步执行的，输出顺序不确定

Group queue（队列组）

当需要多个线程并发执行时，然后等多个线程都结束后，在汇总执行结果，可以使用group queue。

1、使用场景：同时下载多个图片，所有图片下载完成之后去主线程更新UI，或者其他线程队列执行其他任务。

2、多个组队并行执行

// 全局队列
dispatch_queue_t conCunrrentQueue = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0);
NSLog(@"current task");
// 组队列
dispatch_group_t groupQueue = dispatch_group_create();

dispatch_group_async(groupQueue, conCunrrentQueue, ^{
    NSLog(@"并行任务1");
});
dispatch_group_async(groupQueue, conCunrrentQueue, ^{
    NSLog(@"并行任务2");
});

// 该方法可以监听groupQueue中的任务是否全都执行完毕
dispatch_group_notify(groupQueue, dispatch_get_main_queue(), ^{
    NSLog(@"并行任务全部执行完毕");
});
NSLog(@"next task");

控制台输出：

2016-11-26 15:49:17.314 gcd[2550:1113913] current task
2016-11-26 15:49:17.315 gcd[2550:1113913] next task
2016-11-26 15:49:17.315 gcd[2550:1114044] 并行任务1
2016-11-26 15:49:17.315 gcd[2550:1114045] 并行任务2
2016-11-26 15:49:17.321 gcd[2550:1113913] 并行任务全部执行完毕

3、在当前线程阻塞的同步等待dispatch_group_wait

 dispatch_group_t groupQueue = dispatch_group_create();
dispatch_time_t delayTime = dispatch_time(DISPATCH_TIME_NOW, 10 * NSEC_PER_SEC);
dispatch_queue_t conCurrentGlobalQueue = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0);
NSLog(@"current task");
dispatch_group_async(groupQueue, conCurrentGlobalQueue, ^{

    long isExecuteOver = dispatch_group_wait(groupQueue, delayTime);
    if (isExecuteOver) {
        NSLog(@"wait over");
    } else {
        NSLog(@"not over");
    }
    NSLog(@"并行任务1");
});
dispatch_group_async(groupQueue, conCurrentGlobalQueue, ^{
    NSLog(@"并行任务2");
});

控制台输出：

2016-11-26 16:53:51.639 gcd[2653:1164546] current task
2016-11-26 16:53:51.641 gcd[2653:1164709] 并行任务2
2016-11-26 16:54:01.642 gcd[2653:1164712] wait over
2016-11-26 16:54:01.642 gcd[2653:1164712] 并行任务1

dispatch_time(dispatch_time_t when, int64_t delta);

参数注释：第一个参数一般是DISPATCH_TIME_NOW，表示从现在开始
第二个参数是延时的具体时间
延时1秒可以写成如下几种：
NSEC_PER_SEC—-每秒有多少纳秒
dispatch_time(DISPATCH_TIME_NOW, 1NSEC_PER_SEC);
USEC_PER_SEC—-每秒有多少毫秒（注意是指在纳秒的基础上）
dispatch_time(DISPATCH_TIME_NOW, 1000USEC_PER_SEC); //SEC—毫秒
NSEC_PER_USEC—-每毫秒有多少纳秒。
dispatch_time(DISPATCH_TIME_NOW, USEC_PER_SEC*NSEC_PER_USEC);SEC—纳秒
