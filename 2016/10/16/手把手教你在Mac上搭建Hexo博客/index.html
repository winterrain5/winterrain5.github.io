手把手教你在Mac上搭建Hexo博客

发表于 2016-10-16   |   分类于 Hexo   |     |   阅读次数

经过一年的开发后，终于想起来要搭一个博客。百度谷歌了一圈，发现还是Hexo的看着最舒心
接下来就是我一路走的坑，顺便分享给大家。

环境配置

1、Node.js

用来生成静态页面，可在Node.js官网下载，傻瓜式的安装完成后，可用node -v检查是否安装成功

2、Git

用来将本地的Hexo提交到Github上。这里就不赘述如何安装Git了。

安装Hexo

环境配置都准备稳妥后就可以安装Hexo了，终端执行如下命令

$ sudo npm install -g hexo

输入密码开始安装

初始化

终端cd到一个你选定的目录，执行hexo init命令

$ hexo init blog

执行完毕后会在你指定的文件下生成一个文件夹blog。cd到blog，执行如下命令：

$ npm install

接着开启hexo服务器

$ hexo s

此时你的终端应该会有如下信息

shidongdongdeMacBook-Air:blog winterrain$ hexo s
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

在浏览器打开http://localhost:4000/，当你看到这样的页面时，那么恭喜你第一步已经完成了。

关联Github

登录你的Gitbub账号,新建仓库，名为用户名.github.io固定写法，如winterrain5.github.io

本地的blog文件夹的结构是这样的

_config.yml    
db.json 
node_modules 
package.json
scaffolds
source
themes

在终端用vim打开_config.yml,下滑到最后，改成如下：

deploy:
      type: git
      repository: https://github.com/winterrain5/winterrain5.github.io.git
      branch: master

你需要将git地址中的winterrain5改成你的用户名

vim打开_config.yml后，在英文输入法下，按i进入编辑模式，通过方向键移动到你要修改的地方，修改完毕后依次按esc、:、wq后保存并退出编辑

终端中cd到blog文件夹下，执行生成静态页面命令

$ hexo g

若报如下错
ERROR Local hexo not found in ~/blog
ERROR Try runing: 'npm install hexo --save'

则执行命令
npm install hexo --save

再执行配置命令

$ hexo d

若执行hexo d仍然报错：无法连接git或者找不到git，则执行如下命令

$ npm install hexo-deployer-git --save

再次执行hexo g、hexo d

执行成功后，浏览器中打开https://winterrain5.github.io(当然winterran5需要改成你的用户名)看到的页面和http://localhost:4000是一样的

发布文章

接下来就到了激动人心的时刻了，如何发布一篇文章同步到你的博客上
终端cd到blog,执行如下命令新建文章

$ hexo new "articleName"

在/blog/source/_posts下会生成articleName.md文件，你可以打开进行编辑，我用的是Mou编辑器，文章编辑完成后，终端cd到blog文件夹下，执行命令发布

$ hexo g // 生成静态页面

$ hexo d  // 将文章部署到github

至此，Mac上搭建基于Github的Hexo博客就已经完成了，下面介绍安装theme

安装theme

Hexo官网上有很多主题，你可以挑个自己喜欢的。我用的是hexo-theme-next

拷贝链接执行git clone,克隆到 /blog/themes 文件夹下

将_config.yml里theme的名称landscape修改为next

终端执行hexo s后可以先预览本地的效果，感觉满意的话再执行hexo g,hexo d

修改主题样式

修改blog/_config.yml文件和blog/themes/next/_config.yml文件中对应的属性名称即可。 nexT使用文档里有详细的介绍。
