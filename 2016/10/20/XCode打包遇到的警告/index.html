Xcode打包遇到的警告

发表于 2016-10-20   |   分类于 编程工具   |     |   阅读次数

第一次提交版本的时候遇到了各种各样的坑！！！

为了以后少给自己挖坑，总结了一下，也希望能帮到第一次提交版本的童鞋。

ERROR ITMS-90046/90085

“Invalid Code Signing Entitlements. Your application bundle‘s signature contains code signing entitlements that are not supported on iOS. Specifically, value ‘*‘ for key ‘com.apple.

原因: 生成的API分析文件过大，系统不能在提交前，完成API使用信息的校验。

解决 : clean下Xcode缓存，重新打包上传。

ERROR ITMS-90188

“Invaliad Pre-Release Build.The build version (CFBundleVersion) ‘1.0’ in the Info.plist file must be higher than the build version….”

原因: 打包的构建版本号与上次提交的构建版本号相同。

解决 : 每次打包提交前build要往上递增,构建版本号最好要比版本号要高

ERROR ITMS-90086

“missing 64-bit support. beginning on february 1, 2015, new iOS apps submitted to the app store must be include 64-bit support and be built with the ios8 SDK……

原因: 提交的app必须是64位的

解决 : 按如图所示修改

ERROR ITMS-90060

原因: 版本号格式与上次提交的不一致

解决 : 改过来就好咯

ERROR ITMS-4236

原因: 貌似是版本号中带有字符

ERROR ITMS-90098

解决 : 修改Deployment Target为更高版本。未亲测！

ERROR ITMS-90535/90049

原因： 第三方平台的info。plist在捣蛋

解决： 找到直接删掉

使用了Reveal

原因： 打包提交时会提示你使用了私有的API

解决： 每次打包前把Reveal的framework删掉

ERROR ITMS-90062

“This bundle is invalid. The value for key CFBundleShortVersionString [100] in the Info.plist file must contain a higher version than that of the previously approved version [100].”

解决： 提交新版本时 CFBundleVersion 和 CFBundleShortVersionString 都要大于上个版本

后续遇到会继续更新。。。
