OC设计模式（单例）

发表于 2016-12-02   |   分类于 Objective-C   |     |   阅读次数

在开发中经常会用到单例设计模式，目的是为了在程序的整个生命周期中，只会创建一个类的实例对象，而且只要程序不会不被杀死，该实例对象就不会被释放。

作用

  • 保证程序在运行过程中，一个类只有一个实例对象

  • 由于只有一个实例对象，所以节约系统资源

使用场景

  • 资源共享的情况下，避免由于资源操作时导致的性能。如日志文件、应用配置等。

  • 控制资源的情况下，方便资源之间的互相通信。入线程池等。

创建单例

有两种方法创建单例

  • GCD方式创建单例
  • 互斥锁方式创建单例

假设此时有两条线程：线程1和线程2，都在调用shareInstance方法来创建单例，那么线程1运行到if(_instance == nil) 处发现_instance = nil,那么就会创建一个_instance，假设此时线程2也运行到if的判断处，此时线程1还没有创建完成实例对象，所以此时_instance = nil还是成立的，那么线程2又会创建一个实例。
此时就创建了两个实例对象，导致问题出现。

  • 解决方法1：使用dispatch_once

dispatch_once保证在运行过程中只会被运行一次，那么假设此时线程1先执行shareInstance方法，创建一个实例对象，线程2就不会再去执行dispatch_once的代码了，从而保证了只会创建一个对象。

  • 解决方法2：使用互斥锁

假设此时线程1在执行shareInstance方法，那么synchronize大括号内创建单例的代码就会被当做一个任务被加上一把锁。此时线程2若也想执行shareInstance方法，但是看到线程1加的互斥锁，就会进入睡眠模式。等到线程1执行完毕，才会被唤醒，然后去执行上面的代码，但是此时_instance != nil，所以不会再创建新的实例对象了。从而保证只会创建一个实例对象。

但是互斥锁会影响性能，所以最好还是使用GCD方式创建单例
