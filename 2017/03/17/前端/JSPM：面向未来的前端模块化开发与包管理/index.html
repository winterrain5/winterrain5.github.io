JSPM：面向未来的前端模块化开发与包管理

发表于 2017-03-17   |     |   阅读次数

简介

JSPM:jspm.io - Frictionless browser package management是一种渐进式的框架，它使得为JavaScript使用ES6依赖项处理语法成为可能。jspm结合：

Babel,支持使用未来的语言

SystemJS,一个可了解在用的若干封装形态的模块抽象层

jspm位于Babel和SystemJS之上，以一致的语法处理它们，通过npm和GitHub等注册表提供自动化的模块解析。该框架还包括对CSS加载和一个开发服务器的支持。

安装

Node.js 和 npm

首先需要安装Node.js和npm,通过这个教程可以很快的完成安装。安装后，全局路径上就有两个命令：node 和 npm.

jspm

使用管理员权限在全局路径下安装

npm install jspm -g

一路回车即可安装完成，使用 jspm --version 查看当前版本号

如何使用

1、新建文件夹，在当前路径下执行 npm init，完成后该文件夹下会出现package.json,内容如下：

2、在当前文件夹下安装jspm

npm install jspm --save-dev

完成后会出现node_modeules文件夹

3、创建jspm需要的配置文件

jspm init

4、下载资源，如jQuery

jspm install github:components/jquery

下载完毕后，文件名默认是components/jquery,可以这样修改文件名

jspm install jquery=github:components/jquery

这样文件名就是jquery

因为jspm用到了registey,这里列举了一些常用库的简单名字

如果想指定版本也可以这样：

jspm instal jquery@~2.1.0

5、 删除资源

jspm uninstall jquery

6、为项目创建服务器，这样就可以很方便的监视项目文件的变化

  • 首先需要安装browser-sync

    sudo npm install -g browser-sync

  • 当前文件夹下执行命令

    browser-sync start —server —no-notify —files ‘[files]’

[files] 是你需要监视的文件，多个已逗号分隔
