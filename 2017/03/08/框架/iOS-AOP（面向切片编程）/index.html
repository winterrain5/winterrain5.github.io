发表于 2017-03-08   |     |   阅读次数

AOP

AOP: Aspect Oriented Programming ， 面向切片编程

利用AOP可以针对业务处理过程中的切面进行提取，从而使得业务逻辑各个部分之间的耦合降低，提高程序的可重用性，也提高了开发效率。

AOP的目的是将日志记录、性能统计、安全控制、事务处理、异常处理等代码从业务逻辑代码中划分出来，通过对这些行为的分离，我们希望可以将它们独立到非指导业务逻辑的方法中，进而改变这些行为的时候不影响业务逻辑的代码。

Runtime

利用OC的Runtime特性，使用Method Swizzling我们可以给语言做扩展，从而实现AOP。

开发中经常使用if对字典或者数组做安全处理，在代码中大量使用这个操作写多了就会繁琐。

以NSMutableArray为例，为其追加一个新的方法

方法实现

class_addMethod ，先尝试添加originalSelector是为了做一层保护，因为如果这个类没有实现originalSelector，但是其父类实现了，那么class_getInstanceMethod会返回父类的方法。这样method_exchangeImplementations替换的是父类的那个方法，所以先尝试添加originalSelector，如果已经存在，再用method_exchangeImplementations把原方法的实现和新方法的实现给交换一下

Aspects 一个基于OC的AOP框架

它是一个使iOS面向切面编程的库，可以在不改变类或者对象行为的情况下给类或者对象添加代码。你可以在方法执行前/后或者替代原方法执行一段你想执行的代码。可以认为Aspects是一种Method Swizzling，但是它更易用。

它提供了两个简单的API
